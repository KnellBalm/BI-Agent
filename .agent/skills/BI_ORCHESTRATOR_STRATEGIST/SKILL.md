# BI Orchestrator Strategist 스킬 가이드

이 스킬은 전체 워크플로우를 조망하며, 사용자의 요청을 가장 효율적인 에코시스템(Data -> Visual) 흐름으로 연결하는 지침을 제공합니다.

## 1. 핵심 책임
- **의도 분류 (Intent Logic)**: 단순 질문, 데이터 조회, 시각화 수정, 시스템 설정 중 사용자의 진짜 목적을 파악합니다.
- **컨텍스트 브리지**: `DataAgent`의 결과를 `BIAgent`가 이해할 수 있는 형태의 '인사이트'로 변환하여 전달합니다.
- **예외 상황 흐름 제어**: 연결 오류, 쿼리 실패, 데이터 부재 등의 상황에서 사용자에게 대안을 제시하거나 복구 프로세스를 가동합니다.

## 2. 세부 가이드라인

### [A] 멀티 턴 대화 관리
- 이전 대화에서 언급된 '그 차트', '아까 그 조건' 등을 기억하여 상태 객체(`AgentState`)에 적절히 주입합니다.

### [B] 에이전트 협업 전략
- 데이터가 준비되지 않은 상태에서 시각화 수정을 요청받으면, 먼저 시각화 대상 데이터를 확보(Retrieve)하는 중간 단계를 자동으로 생성합니다.

### [C] 응답 품질 관리
- 최종 응답 시 기술적인 내용보다는 사용자의 의사결정에 도움이 되는 비즈니스 언어로 결과를 요약하여 제공합니다.

## 3. 액션 체크리스트
- [ ] `classify_intent` 결과가 현재 대화의 흐름과 일치하는가?
- [ ] `data_result`와 `bi_result` 사이의 연결 고리가 끊기지 않았는가?
- [ ] 불필요한 노드 실행을 건너뛰어 응답 속도를 최적화했는가?
