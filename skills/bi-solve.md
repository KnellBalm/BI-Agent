# bi-solve — BI 분석 오케스트레이터

DA가 비즈니스 문제를 받아 분석 방향을 잡고, 사전 질문지를 생성하며, 소크라테스 또는 Playbook 모드로 분석을 안내한다.

## 이 스킬의 목적

"다음에 뭘 해야 하나"가 막막한 DA를 위해:
1. 문제를 구조화하고
2. 적합한 분석 프레임워크를 제시하며
3. 사전 질문지로 분석 전 맥락을 정리하고
4. 가설 수립부터 검증까지 안내한다

BI 아웃풋(리포트, 대시보드)은 결과이지 목적이 아니다.

---

## 실행 규칙

1. 절대로 Gate를 건너뛰지 않는다
2. 한 번에 하나의 질문만 한다
3. 사전 질문지 파일 생성 없이 분석을 시작하지 않는다
4. 가설 없이 분석을 완료했다고 선언하지 않는다

---

## 5개 Gate 워크플로우

### [G1] 문제 분류

사용자의 문제를 아래 6개 유형 중 하나로 분류한다:

| 유형 | 키워드 | 파일 |
|---|---|---|
| 진단 | "왜", "원인", "이유", "떨어졌나", "올랐나" | `context/playbooks/diagnostic.md` |
| 탐색 | "어떻게 생겼나", "살펴보고", "분석해봐", "파악" | `context/playbooks/exploratory.md` |
| 비교 | "A vs B", "더 나은", "차이", "A/B 테스트" | `context/playbooks/comparative.md` |
| 예측 | "얼마나 될까", "전망", "예측", "앞으로" | `context/playbooks/predictive.md` |
| 결정 | "어떻게 해야", "선택", "결정", "우선순위" | `context/playbooks/decision.md` |
| 모니터링 | "대시보드", "매주 보고", "KPI", "지표 체계" | `context/playbooks/monitoring.md` |

**유형이 불명확할 때:** 소크라테스식으로 한 번에 하나의 질문으로 파악한다.
예: "이 문제가 '왜 바뀌었나'를 묻는 건가요, 아니면 '어떻게 생겼나'를 알고 싶은 건가요?"

### [G2] 환경 체크

"분석에 사용할 데이터 소스가 있나요? (DB 연결, CSV 파일, 대시보드 등)"

- **있음** → G3으로 진행
- **없음** → "데이터 없이도 진행할 수 있습니다. 수동으로 확인 가능한 것부터 시작하고, 필요한 데이터를 어떻게 구할 수 있는지 같이 생각해보겠습니다." → G3으로 진행

### [G3] 프레임워크 승인 + 사전 질문지 생성

해당 Playbook(`context/playbooks/{type}.md`)의 분석 단계를 요약해 보여준다:

예시 출력:
```
이번 문제는 **진단형**으로 분류했습니다.
표준 분석 접근:
1. 지표 분해 — 어느 구성 요소가 변했는지 파악
2. 세그먼트 드릴다운 — 특정 그룹에서만 발생하는지 확인
3. 시계열 패턴 — 언제부터인지 특정
4. 원인 귀속 — 가설 작성 → 데이터로 검증

이 방향으로 진행할까요? (예/수정)
```

**승인 시:** 다음 사전 질문지 파일을 생성한다.

파일명: `context/sessions/YYYY-MM-DD-{문제-슬러그}.md`
(날짜는 오늘 날짜, 슬러그는 문제 핵심 단어로 영문 소문자 하이픈 연결)

내용:
```markdown
# 분석 사전 질문지: {문제 제목}

**날짜:** YYYY-MM-DD
**유형:** {진단/탐색/비교/예측/결정/모니터링}
**상태:** 작성 중

---

## 분석 목적
> (이 분석으로 무엇을 알고 싶은가?)

## 현재 상황
> (지금 어떤 상황이며, 무엇이 이상한가?)

## 보유 데이터
> (어떤 데이터 소스를 사용할 수 있나? 없으면 "없음"이라고 적어도 됩니다)

## 초기 가설
> (원인에 대한 예상이 있다면? 없으면 공란으로 두세요)

## 성공 기준
> (어떤 결과가 나오면 이 분석이 완료됐다고 할 수 있나?)

---

## 분석 결과 (완료 후 작성)

## 핵심 발견

## 결론 및 다음 액션

## 정기화 여부
- [ ] 정기 분석으로 전환 (주기: )
```

파일 생성 후: "파일을 채우고 '시작'이라고 해주세요."
(파일 경로를 사용자에게 알려준다)

**수정 요청 시:** 수정 사항 반영 후 재제시한다.

### [G4] 가설 수립 및 모드 분기

사전 질문지 파일을 읽는다.

**가설 있고 데이터 확보됨 → Playbook 모드:**
- `context/playbooks/{type}.md`의 체크리스트를 단계별로 실행
- 기존 bi-* 스킬을 필요에 따라 호출

**가설 없거나 데이터 불명확 → 소크라테스 모드:**
- `context/socratic_questions/{type}_qs.md`를 로드
- 질문을 한 번에 하나씩 제시
- 사용자가 가설을 발견하면 Playbook 모드로 전환

**기존 bi-* 스킬 활용 가이드:**
- 데이터 연결 필요: `bi-connect` 먼저 실행
- 데이터 탐색: `bi-explore`
- 분석 실행: `bi-analyze`
- 통계 검정: `bi-stats`
- A/B 비교: `bi-ab`
- 시각화: `bi-viz`
- 최종 리포트: `bi-report`

### [G5] 검증 완료

분석 완료 전 확인:

"초기 가설이 확인됐나요, 기각됐나요? 또는 아직 판단하기 어려운가요?"

- **확인 또는 기각** → 세션 파일에 결과를 append하고 완료
  - `context/sessions/index.md`에 한 줄 요약 추가
  - "이 분석을 정기적으로 반복할 필요가 있나요?"
- **판단 불가** → "어떤 부분이 여전히 불확실한가요? 추가로 확인해야 할 가설이 있나요?" → 분석 계속

---

## 세션 파일 완료 처리

분석 완료 시 사전 질문지 파일에 다음을 append한다:

```markdown
---

## 분석 결과

**완료일:** YYYY-MM-DD
**최종 모드:** 소크라테스 → Playbook / Playbook 직접

## 핵심 발견
> (데이터로 확인된 사실)

## 결론 및 다음 액션
> (확인/기각된 가설, 권고 액션)

## 정기화 여부
- [ ] 정기 분석으로 전환 (주기: )
```

`context/sessions/index.md` 테이블에 한 줄 추가:
```
| YYYY-MM-DD | {문제 제목} | {유형} | {결론 한 줄 요약} | {정기화 여부} |
```
