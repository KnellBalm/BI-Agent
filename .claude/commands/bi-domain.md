# /bi-domain — 비즈니스 도메인 컨텍스트 관리

현재 설정된 비즈니스 도메인 정보를 확인하고 관리합니다.

## 실행 절차

### $ARGUMENTS가 없는 경우 (기본: 현황 확인)
1. `context/` 디렉토리의 모든 파일 읽기
2. 각 파일의 작성 완성도 평가 (템플릿 그대로인지 vs 실제 내용인지)
3. 다음 형식으로 요약 출력:

```
## 🏢 비즈니스 컨텍스트 현황

**도메인**: [회사명/팀명] (context/01 기준)
**완성도**: ██████░░░░ 60% (작성된 섹션 / 전체 섹션)

### 설정된 항목 ✅
- [작성된 항목들]

### 미작성 항목 ⚠️
- [아직 템플릿 상태인 항목들]

### 데이터 소스
- [설정된 DB/파일 목록]

### 핵심 KPI ([개수]개 정의됨)
- [KPI 목록 최대 5개]
```

4. 미작성 항목이 있으면 작성 권유 및 해당 파일 안내

### $ARGUMENTS가 "edit"인 경우
- context/ 파일들 중 어떤 것을 수정할지 안내
- 각 파일의 현재 상태 요약

### $ARGUMENTS가 파일명인 경우 (예: "kpis", "data_sources")
- 해당 context 파일의 현재 내용 요약 표시

## 사용법

```
/bi-domain                  # 전체 도메인 현황 확인
/bi-domain edit             # 수정 안내
/bi-domain kpis             # KPI 정의 현황 확인
/bi-domain data_sources     # 데이터 소스 설정 확인
```

## 도메인 완성도 기준

| 완성도 | 기준 | 분석 품질 |
|--------|------|----------|
| 0-30% | context 파일 미작성 | 일반적인 분석 |
| 30-60% | 비즈니스 컨텍스트 + 데이터 소스 작성 | 도메인 인식 분석 |
| 60-80% | KPI 정의까지 작성 | KPI 기반 분석 |
| 80-100% | 전체 파일 작성 완료 | 전문가 수준 분석 |

## 참조 파일
- `context/` 전체

## 사용 MCP 도구
