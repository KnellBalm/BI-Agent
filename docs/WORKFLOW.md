## 1. MCP란?

**MCP(Model Context Protocol)**는 AI 모델(Claude 등)이 외부 도구·데이터 소스와 표준화된 방식으로 통신하도록 설계된 오픈 프로토콜입니다. 쉽게 말해, AI 채팅창에서 자연어로 요청하면 MCP 서버가 실제 데이터 작업을 대신 수행하고 결과를 돌려줍니다.

bi-agent는 이 MCP 위에 구축된 BI 분석 엔진으로, 98개의 분석 도구를 제공합니다. 사용자는 **도구를 직접 호출할 필요가 없습니다**. AI에게 자연어로 요청하면 AI가 상황에 맞는 도구를 자동으로 선택하고 실행합니다.

```
사용자: "지난 달 매출 상위 10개 상품 알려줘"
  └─> AI가 자동으로: connect_db → run_query → revenue_analysis → generate_report 순서로 도구 호출
  └─> 사용자에게: 분석 결과 + 인사이트 반환
```

## 2. 기본 워크플로우

bi-agent 사용의 기본 흐름은 4단계입니다.

```
[연결] → [탐색] → [분석] → [리포트]
```

### 1단계: 연결

데이터 소스에 연결합니다. 처음 한 번만 설정하면 이후 대화에서 자동으로 재사용됩니다.

```
"prod PostgreSQL DB에 연결해줘. 호스트는 db.company.com, DB명은 analytics야"
"sales.csv 파일 불러와서 분석하고 싶어"
"GA4 데이터 가져오려면 어떻게 해야 해?"
```

### 2단계: 탐색

연결된 데이터의 구조와 내용을 파악합니다.

```
"어떤 테이블이 있어?"
"orders 테이블 구조 보여줘"
"user_events 테이블에 어떤 데이터가 있는지 샘플 보여줘"
```

### 3단계: 분석

목적에 맞는 분석을 요청합니다.

```
"지난 분기 매출 트렌드 분석해줘"
"사용자 이탈 원인을 RFM으로 세그먼트별로 파악해줘"
"A/B 테스트 결과가 통계적으로 유의미한지 검정해줘"
```

### 4단계: 리포트

분석 결과를 원하는 형식으로 정리합니다.

```
"이 분석 결과를 마크다운 리포트로 정리해줘"
"Chart.js 인터랙티브 대시보드로 만들어줘"
"Tableau에서 바로 열 수 있는 TWBX 파일로 내보내줘"
```

## 3. 클라이언트별 사용법

bi-agent는 MCP를 지원하는 모든 AI 클라이언트에서 동작합니다.

### Claude Desktop

1. `~/.claude/claude_desktop_config.json`에 bi-agent MCP 서버 설정 추가
2. Claude Desktop 재시작
3. 채팅창에서 바로 자연어로 분석 요청

```
# 예시 입력
"prod DB에 연결하고 지난 7일간 신규 가입자 수 알려줘"
```

### Cursor

1. `.cursor/mcp.json`에 bi-agent MCP 서버 설정 추가
2. Cursor 재시작
3. **Composer** (`Cmd+I` / `Ctrl+I`) 또는 채팅 패널에서 자연어로 요청

```
# Composer에서 예시
@bi-agent analytics DB에 연결해서 월별 MAU 트렌드 분석해줘
```

### VSCode (GitHub Copilot Chat)

1. `.vscode/mcp.json`에 bi-agent MCP 서버 설정 추가
2. VSCode 재시작
3. **GitHub Copilot Chat** 패널(`Ctrl+Shift+I`)에서 `@mcp` 또는 직접 자연어로 요청

```
# Copilot Chat에서 예시
지난 달 캠페인별 전환율 비교해줘
```

### Antigravity

1. `~/.gemini/antigravity/mcp_config.json`에 bi-agent MCP 서버 설정 추가
2. Antigravity 재시작
3. 대화 모드에서 자연어로 분석 요청 — 여러 단계의 분석을 연속으로 진행 가능

```
# 대화 예시
사용자: BigQuery에 있는 events 테이블 분석해줘
AI: 연결하겠습니다. 어떤 분석이 필요하신가요?
사용자: 최근 30일 퍼널 분석이 필요해
```

> **설정 파일 자동 생성**: `bi-agent setup` 명령어를 실행하면 대화형 마법사가 클라이언트별 설정 파일을 자동으로 생성합니다.

## 4. 분석 시나리오 예시

### 시나리오 1: 매출 분석

지난 분기 매출 현황과 성장률을 파악합니다.

```
1. "analytics DB에 연결해줘"
2. "orders 테이블에서 지난 분기 매출 상위 10개 상품 분석해줘"
3. "각 상품의 전분기 대비 성장률도 계산해줘"
4. "파레토 법칙(80/20) 기준으로 핵심 상품군 구분해줘"
5. "이 결과를 마크다운 리포트로 정리해줘"
```

### 시나리오 2: 이탈 분석

최근 3개월간 이탈한 사용자의 특성을 분석합니다.

```
1. "최근 3개월간 이탈한 사용자(90일 미접속) 추출해줘"
2. "RFM 분석으로 이탈 사용자를 세그먼트별로 나눠줘"
3. "각 세그먼트별 LTV와 평균 주문 금액 비교해줘"
4. "이탈 위험도가 높은 현재 활성 사용자 목록도 뽑아줘"
5. "분석 결과를 기반으로 재활성화 전략 추천해줘"
```

### 시나리오 3: A/B 테스트 분석

버튼 색상 A/B 테스트 결과의 통계적 유의성을 검증합니다.

```
1. "ab_test_results 테이블 구조 보여줘"
2. "실험군(B)과 대조군(A)의 전환율 차이 계산해줘"
3. "이 결과가 통계적으로 유의미한지 카이제곱 검정해줘 (유의수준 0.05)"
4. "실험 결과의 신뢰구간도 계산해줘"
5. "연령대/디바이스별로 세그먼트 이질 효과(HTE) 있는지 확인해줘"
```

### 시나리오 4: 리텐션 분석

지난달 신규 가입자의 7일 리텐션을 추적합니다.

```
1. "지난달 신규 가입자 코호트 기준으로 7일 리텐션 분석해줘"
2. "Day 1, Day 3, Day 7 리텐션 곡선 데이터 뽑아줘"
3. "리텐션이 높은 사용자와 낮은 사용자의 초기 행동 패턴 비교해줘"
4. "리텐션 개선을 위한 핵심 액션 포인트 추천해줘"
5. "결과를 Chart.js 인터랙티브 대시보드로 만들어줘"
```

### 시나리오 5: 멀티소스 전환율 분석

GA4 트래픽 데이터와 DB 매출 데이터를 결합하여 전환율을 분석합니다.

```
1. "GA4에 연결해서 지난 30일 채널별 세션 수 가져와줘"
2. "analytics DB에서 같은 기간 채널별 실제 구매 건수도 뽑아줘"
3. "두 데이터를 결합해서 채널별 전환율(구매/세션) 계산해줘"
4. "전환율이 낮은 채널의 퍼널 이탈 지점 분석해줘"
5. "채널별 CAC와 ROAS도 함께 계산해서 종합 리포트 만들어줘"
```

## 5. 멀티 소스 분석

bi-agent는 여러 데이터 소스를 동시에 연결하고 결합할 수 있습니다.

### 여러 연결 ID 사용

각 데이터 소스는 고유한 `conn_id`로 식별됩니다. 여러 소스를 동시에 연결할 수 있습니다.

```
# 여러 DB 동시 연결
"prod DB에 'prod'라는 이름으로 연결하고, analytics DB에 'analytics'라는 이름으로도 연결해줘"

# 각각 다른 소스에서 데이터 조회
"prod DB에서 주문 데이터 가져오고, analytics DB에서 이벤트 로그 가져와서 비교해줘"
```

### 크로스 소스 조인

서로 다른 소스의 데이터를 DuckDB로 조인합니다.

```
# 파일 + DB 결합
"user_segments.csv 파일의 세그먼트 정보와 prod DB의 구매 데이터를 user_id 기준으로 조인해서 분석해줘"

# DB + GA4 결합
"GA4에서 트래픽 데이터 가져오고, analytics DB 매출 데이터랑 결합해서 채널별 전환율 계산해줘"

# 두 DB 비교
"prod DB와 staging DB의 동일 쿼리 결과를 비교해서 데이터 정합성 확인해줘"
```

### connections.json으로 자동 연결 (자동 설정)

`connections.json` 파일에 미리 정의해두면 매번 연결 정보를 입력하지 않아도 됩니다.

```json
{
  "connections": [
    {
      "id": "prod",
      "type": "postgresql",
      "host": "db.company.com",
      "dbname": "production"
    },
    {
      "id": "analytics",
      "type": "bigquery",
      "project_id": "my-project"
    }
  ]
}
```

## 6. 트러블슈팅 FAQ

### Q: 도구가 호출이 안 돼요

**A**: 다음을 순서대로 확인하세요.

1. MCP 서버가 클라이언트 설정 파일에 올바르게 등록되어 있는지 확인
2. 클라이언트를 완전히 종료했다가 재시작
3. 터미널에서 `python -m bi_agent_mcp` 직접 실행해서 오류 메시지 확인
4. `bi-agent setup` 명령어로 설정 마법사 재실행

```bash
# 설치 확인
python -c "from bi_agent_mcp.server import mcp; print(len(mcp._tool_manager._tools), '개 도구 로드됨')"
```

### Q: 연결이 실패해요

**A**: 연결 오류의 주요 원인:

- 호스트명/포트 오타: DB 접속 정보 재확인
- 방화벽/VPN: 사내 DB는 VPN 연결 후 시도
- 자격증명 만료: 비밀번호 변경 여부 확인
- 환경 변수 미설정: `BI_AGENT_PG_HOST` 등 환경 변수가 MCP 서버에 전달되는지 확인

```bash
# 환경 변수 확인
env | grep BI_AGENT_
```

### Q: SQL 결과가 이상해요

**A**: bi-agent는 기본 500행 제한이 있습니다. 전체 데이터가 필요하면:

```
"QUERY_LIMIT 늘려줘" 또는 "결과 행 제한 없이 전체 데이터 가져와줘"
```

또는 환경 변수 `BI_AGENT_QUERY_LIMIT`을 늘려서 MCP 설정에 추가하세요.

> 주의: SELECT 이외의 DML 쿼리(INSERT, UPDATE, DELETE)는 보안상 자동 차단됩니다.

### Q: 파일을 분석하려면?

**A**: CSV, Excel 파일은 직접 불러올 수 있습니다.

```
"~/Downloads/sales_data.csv 파일 불러와서 분석해줘"
"reports/q1_2026.xlsx 엑셀 파일 로드하고 컬럼 구조 보여줘"
```

파일 로드 후 SQL로 쿼리하거나 BI 분석 도구를 바로 적용할 수 있습니다.

### Q: 설정을 바꾸려면?

**A**: 두 가지 방법이 있습니다.

```bash
# 방법 1: 설정 마법사 재실행
bi-agent setup

# 방법 2: 클라이언트 설정 파일 직접 편집
# Claude Desktop: ~/.claude/claude_desktop_config.json
# Cursor: .cursor/mcp.json
# VSCode: .vscode/mcp.json
```

설정 변경 후에는 반드시 클라이언트를 재시작해야 적용됩니다.
