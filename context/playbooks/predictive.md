# 예측 (Predictive) 분석 Playbook

> "앞으로 어떻게 될까?" 유형의 문제에 적용한다.

## 언제 사용하나

- 다음 달/분기 지표를 예측해야 할 때
- 트렌드가 계속될지 판단해야 할 때
- 목표 달성 가능성을 추정해야 할 때

## 표준 분석 단계

### 1단계: 예측 목적 및 범위 정의
- 무엇을 예측하는가? (지표, 기간, 정확도 요구 수준)
- 예측이 어떤 결정에 쓰이는가?
- 허용 오차 범위 (±10%? ±30%?)

### 2단계: 과거 데이터 탐색
- 사용 가능한 데이터 기간
- 계절성 패턴 (요일, 월별)
- 트렌드 방향 (상승/하강/보합)
- 이상치 및 구조 변화 시점

### 3단계: 예측 방법 선택
**데이터 적을 때 (< 1년):** 이동평균, 선형 추세
**데이터 충분할 때 (> 1년, 계절성 있음):** 계절 분해, Holt-Winters
**정교한 예측이 필요할 때:** Prophet, ARIMA (데이터 사이언티스트 필요)

### 4단계: 예측 실행 및 검증
- 최근 데이터 일부를 검증 세트로 보류
- 모델을 과거 데이터로 학습
- 검증 세트에서 오차(MAPE) 측정

### 5단계: 결과 제시
- 점 예측(단일 값) + 예측 구간(범위)
- "X가 Y가 될 것이다"보다 "X는 Y~Z 범위일 가능성이 높다"
- 예측을 무너뜨릴 가정 및 리스크 명시

## 체크리스트

- [ ] 예측 지표와 기간 명시
- [ ] 사용 가능한 데이터 기간 확인
- [ ] 계절성/트렌드 패턴 확인
- [ ] 예측 방법 선택 및 이유 명시
- [ ] 예측 구간(범위) 포함
- [ ] 주요 가정 및 리스크 목록

## 흔한 함정

- **과적합**: 과거 패턴에 너무 맞춰 미래를 예측하면 실제와 괴리
- **구조 변화 무시**: 비즈니스가 바뀌었으면 과거 데이터가 미래를 설명하지 못할 수 있음
- **점 예측만 제시**: "내년 매출은 10억"보다 "9~11억 범위"가 정직하고 유용

## 도메인별 변형

**SaaS MRR 예측:** 신규/이탈/확장 MRR 각각 예측 후 합산
**마케팅 예산 예측:** CAC × 목표 고객 수 = 필요 예산
**재고/운영:** 수요 예측 → 안전 재고 계산
