# 결정 (Decision) 분석 Playbook

> "어떻게 해야 하나?" 유형의 문제에 적용한다.

## 언제 사용하나

- 여러 옵션 중 하나를 선택해야 할 때
- 투자/실행 여부를 판단해야 할 때
- 우선순위를 정해야 할 때

## 표준 분석 단계

### 1단계: 결정 프레임 설정
- 결정의 주체는 누구인가?
- 결정의 가역성: 되돌릴 수 있는가? (단방향 문은 더 신중하게)
- 결정 기한

### 2단계: 옵션 목록 작성
- 고려 중인 옵션 나열 (최소 3개 — "현상 유지"도 옵션)
- 각 옵션의 예상 결과

### 3단계: 평가 기준 설정
- 비즈니스 임팩트 (정량: 매출, 비용, 효율)
- 실행 가능성 (시간, 리소스)
- 리스크 (최악의 경우 시나리오)
- 기회비용 (이것을 하면 못 하는 것은?)

### 4단계: 옵션 평가
각 옵션을 기준별로 평가:
- 임팩트 크기 (크다/보통/작다)
- 확신도 (높다/중간/낮다)
- 실행 속도 (빠르다/보통/느리다)

### 5단계: 권고 및 다음 스텝
- 권고 옵션 1개와 이유
- 핵심 가정 (이 가정이 틀리면 권고가 바뀜)
- 실행 전 검증해야 할 것 (있다면)

## 체크리스트

- [ ] 결정 기한 명시
- [ ] 옵션 목록 (최소 3개, 현상 유지 포함)
- [ ] 평가 기준 3개 이상
- [ ] 각 옵션 평가 완료
- [ ] 권고 옵션 1개와 근거
- [ ] 핵심 가정 목록

## 흔한 함정

- **확증 편향**: 이미 마음에 있는 옵션을 지지하는 데이터만 찾음. 반증 데이터도 적극 탐색
- **옵션 누락**: "현상 유지"를 옵션으로 포함하지 않음
- **결정 회피**: 데이터가 불완전할 때 결정을 미룸. 불확실성 하에서의 최선 결정이 목표

## 도메인별 변형

**프로덕트 로드맵 우선순위:** Impact/Effort 매트릭스 + 고객 데이터 근거
**마케팅 채널 투자:** LTV/CAC 기준 + 스케일 가능성
**가격 정책:** 탄력성 추정 + 경쟁사 벤치마크
