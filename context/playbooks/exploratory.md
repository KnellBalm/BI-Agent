# 탐색 (Exploratory) 분석 Playbook

> "X가 어떻게 생겼나?" 유형의 문제에 적용한다.

## 언제 사용하나

- 새로운 데이터셋을 처음 접할 때
- "이 지표가 어떤 모습인지" 알고 싶을 때
- 가설 없이 패턴을 발견하고 싶을 때
- 후속 분석을 위한 준비 단계로

## 표준 분석 단계

### 1단계: 데이터 품질 확인
- 결측값 비율 및 분포
- 이상값 (outlier) 존재 여부
- 데이터 기간 및 갱신 주기

### 2단계: 기술 통계
- 평균, 중앙값, 표준편차
- 최솟값, 최댓값, 분위수
- 분포 형태 (정규, 왜도, 첨도)

### 3단계: 분포 시각화
- 히스토그램 / 박스플롯
- 시계열 추세선
- 카테고리별 분포 비교

### 4단계: 패턴 및 상관 탐색
- 주요 지표 간 상관관계
- 시간적 패턴 (요일, 월별, 계절성)
- 세그먼트 간 차이

### 5단계: 이상치 및 주목할 점
- 예외적으로 높거나 낮은 값
- 기대와 다른 패턴
- 추가 분석이 필요한 가설

## 체크리스트

- [ ] 데이터 기간 및 행 수 확인
- [ ] 결측값 비율 확인 (10% 이상이면 주석)
- [ ] 핵심 지표 기술 통계 완료
- [ ] 분포 시각화 1개 이상
- [ ] 주목할 패턴 3개 이상 기록
- [ ] 후속 분석 가설 또는 질문 목록 작성

## 흔한 함정

- **평균의 함정**: 평균만 보고 분포를 놓침. 반드시 중앙값, 분위수 함께 확인
- **탐색 = 결론 아님**: EDA는 가설을 만드는 단계. "발견했다"가 아니라 "확인해야 할 것 같다"
- **데이터 품질 무시**: 결측값/이상값 처리 방식이 결론에 큰 영향

## 도메인별 변형

**SaaS:** 주요 탐색 대상 = 코호트별 리텐션 곡선, 기능별 사용 빈도, 구독 플랜 분포
**커머스:** 주요 탐색 대상 = 카테고리별 구매 빈도, 객단가 분포, 재구매 간격
**B2B:** 주요 탐색 대상 = 딜 사이즈 분포, 영업 사이클 길이, 채널별 리드 품질
